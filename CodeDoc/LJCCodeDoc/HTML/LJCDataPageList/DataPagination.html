Remarks

The Pagination Initial Load Process

The Form that includes the LJCDataGrid control, contains a private DataManager object. This DataManager object is initialized and the PageSize set before the LJCDataGrid columns are configured.

The LJCDataGrid columns must be configured before attempting to display any data in the LJCDataGrid control.

The Form also includes a private Pagination object. This object is initialized before attempting to display any data in the LJCDataGrid control.

After an instance of the Pagination object is created, an Event Handler is created for the Pagination.DataRetrieve event and the SelectCurrentRow event. Then the Pagination.PageSize is set and the Pagination.Initialize() method is called.

The Pagination.Initialize() method fires the Pagination.DataRetrieve event. The DataRetrieve event handler saves the Pagination.PageStartIndex and PageSize to the DataManager properties of the same name. It then calls the List Form private DataRetrieve() method.

The List Form DataRetrieve() method calls the DataManager.Load() method. This method uses the DataManager PageStartIndex and PageSize values to load the next or previous pages of data. The DataRetrieve() method shows the first page of data for the initial load.

The Paging Process

The PageGridProcess() method is called when the user changes the selected item in the list.

The PageGridProcess() method calls the Pagination.PageUp() method if the selected record is in the top 10 records of the list and pagination is not at the beginning of the data. The Pagination FirstRowIndex and CurrentRowIndex are saved from the current position of the grid.

The PageGridProcess() method calls the Pagination.PageDown() method if the selected record is in the last 10 of the list and pagination is not at the end of the data. The Pagination.FirstRowIndex, CurrentRowIndex and RowCount are set from the current position and content of the grid.

As the user pages through the data, the Pagination PageUp() and PageDown() methods include the old previous or next page of data so that two pages of data are displayed. This provides the user with better data context as they can always see the rows of data on both sides of a page break.

The Pagination PageUp() and PageDown() methods also calculate the new FirstRowIndex and CurrentRowIndex values. Then they call the form SelectCurrentRow event which sets those values. This prevents the list from appearing to move which preserves the current item context and provides a smoother visual experience for the user.
